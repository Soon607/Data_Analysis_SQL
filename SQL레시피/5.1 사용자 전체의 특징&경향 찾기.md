# 1. 사용자의 액션 수 집계하기
```sql
with base (
select
  action, -- 어떤 행동을 했는지
  user_id, -- 누가 했는지
  session, -- 어떤 방문(세션)에서 했느지
  count(*) as action_cnt -- '동일 세션' 내에서 해당 '액션'이 몇 번 발생했는지 합산
from action_log
group by action,user_id,session)
```
1) **노이즈**와 **진짜 행동**의 구분

   * 단순 `count(*)`를 하면 구매 의사가 엄청 높은 것처럼 왜곡될 수 있다.
   
   * 이 쿼리를 통해 **이 세션에서 장바구니 담기를 3번 했구나**와 같은 정보(action_cnt)를 유지하면서도, 전체 통계를 낼 때 1개의 세션으로 깔끔하게 처리할 수 있다.


2) **계산**의 편의성
   * 액션별 평균 사용률은? --> base에서 action으로 `group by`하여 계산

   * 한 세션에서 특정 액션을 가장 많이 한 유저는? --> action_cnt가 가장 높은 user_id 추출

```sql
select
  action,
  count(distinct user_id) as user_count, -- 1. 유니크 사용자 수
  round(avg(action_cnt)2,) as avg_count_per_session, -- 2. 세션당 평균 발생 횟수 (Count per User/Session)
  round(100.0*count(distinct user_id)/(select count(distinct user_id) from action_log,2) as usage_rate -- 3. Usage Rate
  -- (특정 액션을 한 유저 수 / 서비스 전체 유저 수) *100
from base
group by action;
```
1) **"얼마나 많은 사람이?"**

   * 해당 액션을 최소 한 번이라도 수행한 **순수 사용자**
   
   * 단순 로그 개수가 아닌 **사람 수**를 세기 때문에 규모를 파악하기에 가장 정확한 지표
   
2) **"한 번 올 때 얼마나 집중적으로?"**

   * 사용자가 한 세션(한 번의 방문) 동안 해당 액션을 평균적으로 몇 번이나 반복했는지를 보여준다.
   
   * 이 수치가 높다면 해당 기능이 **중독성이** 있거나, 사용자가 **목적을 달성하기 위해 여러 번 조작**해야 함을 뜻한다.
   
3) **전체 중 몇 %나?**

   * 서비스에 접속한 전체 인원 중 이 액션을 경험한 사람의 비율
